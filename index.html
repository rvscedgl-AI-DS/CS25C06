Boolean Algebra & K-Maps

Beginner

Boolean Algebra Cheat Sheet

Basic Laws
Identity: A+0=A, A·1=A
Null: A+1=1, A·0=0
Idempotent: A+A=A, A·A=A
Inverse: A+A'=1, A·A'=0
Commutative Law
A+B = B+A, A·B = B·A
Associative Law
(A+B)+C = A+(B+C), (A·B)·C = A·(B·C)
Distributive Law
A·(B+C) = A·B + A·C, A+(B·C) = (A+B)·(A+C)
De Morgan's Theorem
(A+B)' = A'·B', (A·B)' = A'+B'

K-Map Simplification Steps

1 Create K-Map Grid
For n variables, create 2ⁿ cells. Label rows/columns with Gray code.
2 Fill Truth Table Values
Transfer 1's and 0's from truth table to corresponding cells.
3 Group Adjacent 1's
Group in powers of 2 (1,2,4,8...). Groups can wrap around edges.
4 Write Product Terms
For each group, write product term for variables that don't change.
5 Sum the Product Terms
OR all product terms to get simplified Boolean expression.

Quick Notes

K-Map Grouping Rules: Groups must be rectangular, size must be power of 2, maximize group size, minimize number of groups.
Don't Care Conditions (X): Can be treated as 0 or 1 to maximize grouping.
POS vs SOP: For Product of Sums, group 0's instead of 1's and complement the result.

Common Simplifications

A + A'B = A + B (Absorption Theorem)
AB + A'C + BC = AB + A'C (Consensus Theorem)
(A+B)(A'+C)(B+C) = (A+B)(A'+C) (Dual Consensus)

Logic Gates & Circuits

Beginner

Logic Gates Reference

AND Gate
Output=1 only if ALL inputs=1. Symbol: &, Expression: Y=A·B
OR Gate
Output=1 if ANY input=1. Symbol: ≥1, Expression: Y=A+B
NOT Gate
Inverts input. Symbol: △+○, Expression: Y=A'
NAND Gate
AND followed by NOT. Universal gate. Y=(A·B)'
NOR Gate
OR followed by NOT. Universal gate. Y=(A+B)'
XOR Gate
Output=1 if inputs are different. Symbol: =1, Y=A⊕B
XNOR Gate
Output=1 if inputs are same. Y=A⊙B, Complement of XOR

Universal Gates Implementation

1 Using NAND as NOT
Connect both inputs together: NOT A = NAND(A,A)
2 NAND as AND
AND(A,B) = NOT(NAND(A,B)) = NAND(NAND(A,B), NAND(A,B))
3 NAND as OR
OR(A,B) = NAND(NOT A, NOT B) = NAND(NAND(A,A), NAND(B,B))
4 NOR as NOT
NOT A = NOR(A,A)
5 Building Any Circuit
Convert AND/OR/NOT to NAND/NOR equivalents using De Morgan's

Gate Properties

Fan-out: Maximum number of gates a gate output can drive (typically 10-20 for TTL).
Propagation Delay: Time between input change and output response (ns range).
Power Dissipation: CMOS gates consume power only during switching.
IC Packages: DIP (Dual In-line Package) most common for prototyping.

Number Systems & Arithmetic

Intermediate

Number Systems Quick Reference

Binary (Base-2)
Digits: 0,1. Weight: 2ⁿ. Example: 1011₂ = 11₁₀
Octal (Base-8)
Digits: 0-7. Weight: 8ⁿ. Example: 75₈ = 61₁₀
Decimal (Base-10)
Digits: 0-9. Weight: 10ⁿ. Example: 123₁₀
Hexadecimal (Base-16)
Digits: 0-9, A-F. Weight: 16ⁿ. Example: 1F₆ = 31₁₀
Conversion Shortcuts
Binary↔Hex: Group 4 bits = 1 hex digit
Binary↔Octal: Group 3 bits = 1 octal digit
Decimal→Binary: Repeated division by 2

Signed Number Arithmetic

1 Sign-Magnitude
MSB=sign (0=+,1=-), remaining bits=magnitude. Range: -(2ⁿ⁻¹-1) to +(2ⁿ⁻¹-1)
2 1's Complement
Negative = bitwise complement of positive. Has -0 and +0 problem.
3 2's Complement
Negative = 1's complement + 1. Most common. Range: -2ⁿ⁻¹ to +(2ⁿ⁻¹-1)
4 Addition/Subtraction
Use 2's complement: A-B = A + (2's complement of B). Ignore final carry.
5 Overflow Detection
Overflow occurs when sign bits of operands are same but result sign is different.

Key Formulas

Decimal to Binary: N₁₀ = bₙ2ⁿ + bₙ₋₁2ⁿ⁻¹ + ... + b₁2¹ + b₀2⁰
2's Complement: -X = (NOT X) + 1
Range for n-bit 2's complement: -2ⁿ⁻¹ to 2ⁿ⁻¹-1

Memory Aid

Binary to Hex: "1-2-4-8" - These are the weights of the 4 binary bits that make one hex digit.

2's Complement: "Flip and Add 1" - To get negative of a number in 2's complement.

Computer Architecture

Intermediate

Architecture Comparison

Von Neumann
Single memory for data & instructions. Sequential execution. Bottleneck: von Neumann bottleneck.
Harvard
Separate memories for data & instructions. Parallel fetch possible. Used in DSPs, microcontrollers.
RISC (Reduced)
Simple instructions, fixed length, load/store architecture, many registers, hardwired control.
CISC (Complex)
Complex instructions, variable length, memory-to-memory operations, microprogrammed control.

Performance Metrics

1 MIPS Calculation
MIPS = Clock Rate / (CPI × 10⁶) = Instruction Count / (Execution Time × 10⁶)
2 CPI Formula
CPI = Clock Cycles / Instruction Count = Σ(CPIᵢ × Iᵢ) / Instruction Count
3 CPU Time
CPU Time = Instruction Count × CPI × Clock Cycle Time
4 Speedup
Speedup = Old Execution Time / New Execution Time = Old CPI / New CPI (if clock same)
5 Amdahl's Law
Overall Speedup = 1 / [(1 - F) + F/S] where F=fraction improved, S=speedup of that fraction
Feature RISC CISC
Instruction Size Fixed (4 bytes typical) Variable (1-15 bytes)
Registers Many (32-128) Fewer (8-16)
Memory Access Load/Store only Memory-to-memory allowed
Control Unit Hardwired (faster) Microprogrammed (flexible)
Examples ARM, MIPS, RISC-V x86, IBM 360, VAX

ALU Design & Adders

Advanced

Adder Types & Characteristics

Half Adder
Adds 2 bits. Outputs: Sum = A⊕B, Carry = A·B. No carry input.
Full Adder
Adds 3 bits (A,B,Cin). Sum = A⊕B⊕Cin, Cout = AB + BCin + ACin.
Ripple Carry Adder
n full adders cascaded. Slow (O(n) delay). Simple design.
Carry Lookahead Adder
Fast (O(log n) delay). Uses generate (G) and propagate (P) signals.
Carry Select Adder
Calculates for carry=0 and carry=1 simultaneously, then selects. Faster but more hardware.

Booth's Algorithm Steps

1 Initialize
A=0, Q=Multiplicand, M=Multiplier, Q₋₁=0, count=n
2 Check Q₀Q₋₁
If 01: A = A + M
If 10: A = A - M
If 00 or 11: Do nothing
3 Arithmetic Shift Right
Shift A, Q, Q₋₁ right by 1 bit. MSB of A remains same (sign extension).
4 Decrement Count
count = count - 1. If count > 0, repeat from step 2.
5 Result
Product is in A and Q registers combined.

Adder Formulas

Full Adder: S = A ⊕ B ⊕ Cin, Cout = AB + BCin + ACin
Carry Lookahead: Cᵢ₊₁ = Gᵢ + Pᵢ·Cᵢ where Gᵢ = Aᵢ·Bᵢ, Pᵢ = Aᵢ ⊕ Bᵢ
Booth Recoding: 0→0, 1→+1, 2→0 with carry, etc. for signed multiplication

Booth's Algorithm Trick

"01 means add, 10 means subtract, 00/11 just shift" - Remember this for Booth's algorithm steps.

For CLA: "Generate means both 1's, Propagate means exactly one 1"

Pipelining & Hazards

Advanced

Pipeline Hazards

Structural Hazards
Hardware resource conflict. Solution: Duplicate resources, stall pipeline.
Data Hazards
Data dependency between instructions. Types: RAW, WAR, WAW.
Control Hazards
Branch instructions change PC. Solutions: Prediction, delayed branch.
Hazard Solutions
Forwarding: Bypass ALU result to next instruction
Stalling: Insert bubbles (NOPs)
Reordering: Compiler optimization

5-Stage Pipeline (MIPS)

IF Instruction Fetch
Fetch instruction from memory using PC. Update PC to PC+4.
ID Instruction Decode
Decode instruction, read registers, sign-extend immediate, detect hazards.
EX Execute
Perform ALU operation, calculate branch target, resolve data hazards.
MEM Memory Access
Load/store from/to memory. For branches: update PC if taken.
WB Write Back
Write result to register file (ALU result or loaded data).
Hazard Type Detection Solution Hardware/Software
RAW (Read After Write) Check if source reg = dest reg of previous instr Forwarding, Stalling Hardware
WAR (Write After Read) Check if dest reg = source reg of later instr Register renaming Hardware/Compiler
WAW (Write After Write) Check if dest reg = dest reg of previous instr Register renaming, in-order completion Hardware
Control/Branch Branch instruction in pipeline Prediction, delayed slot Hardware/Compiler

Pipeline Speedup

Speedup = (Non-pipelined time) / (Pipelined time) = (n × k) / (n + k - 1)
where n = number of instructions, k = number of pipeline stages
Ideal speedup = k (when n >> k)

Memory Systems

Intermediate

Memory Hierarchy

Registers
Fastest, smallest (KB), inside CPU, SRAM technology
Cache (L1/L2/L3)
Fast (ns), small (KB-MB), SRAM, managed by hardware
Main Memory (RAM)
Medium speed (10-100ns), GB size, DRAM, volatile
Secondary Storage
Slow (ms), large (TB), HDD/SSD, non-volatile
Cache Mapping
Direct: 1 block → 1 line (simple, conflicts)
Associative: Any block → any line (complex, no conflicts)
Set-associative: n-way compromise

Cache Replacement Policies

1 LRU (Least Recently Used)
Replace block not used for longest time. Best practical performance.
2 FIFO (First In First Out)
Replace oldest block. Simple but suffers from Belady's Anomaly.
3 Random
Replace random block. Simple hardware, acceptable performance.
4 LFU (Least Frequently Used)
Replace least frequently accessed. Tracks frequency counters.
5 Optimal (Belady's)
Replace block not used for longest future time. Theoretical optimal.

Memory Technology Notes

SRAM vs DRAM: SRAM faster, more expensive, less dense (6T cell), no refresh needed. DRAM slower, cheaper, denser (1T+1C cell), needs refresh.
ROM Types: PROM (programmable once), EPROM (UV erasable), EEPROM (electrically erasable), Flash (block erasable EEPROM).
HDD vs SSD: HDD has moving parts, sequential access faster, cheaper per GB. SSD no moving parts, random access faster, more durable.

Cache Performance

Average Memory Access Time = Hit Time + Miss Rate × Miss Penalty
Miss Rate = Number of misses / Number of accesses
AMAT for multilevel cache: AMAT = Hit Time₁ + Miss Rate₁ × (Hit Time₂ + Miss Rate₂ × Miss Penalty₂)

I/O Systems & Buses

Intermediate

I/O Techniques

Programmed I/O
CPU polls device status. Simple but inefficient (CPU waits).
Interrupt-Driven I/O
Device interrupts CPU when ready. Efficient but overhead per transfer.
DMA (Direct Memory Access)
Special controller transfers data directly. CPU only involved at start/end.
Bus Types
System Bus: CPU-Memory (fastest)
I/O Bus: Connects peripherals (PCI, USB)
Backplane Bus: Connects boards in system

Interface Standards

PCI Peripheral Component Interconnect
Parallel bus, 32/64 bit, 33/66 MHz, plug-and-play, for internal cards.
USB Universal Serial Bus
Serial, hot-pluggable, versions: 1.0 (1.5 Mbps), 2.0 (480 Mbps), 3.0 (5 Gbps).
SATA Serial ATA
Serial replacement for PATA, for storage devices, 1.5/3/6 Gbps versions.
PCIe PCI Express
Serial, point-to-point, lanes (x1, x4, x8, x16), replaced PCI/AGP.
Bus Characteristic Description Examples
Clock Synchronous Uses clock for timing PCI, System Bus
Asynchronous Uses handshaking signals USB (partially)
Width Number of data lines 8, 16, 32, 64 bits
Bandwidth Data transfer rate USB 3.0: 5 Gbps, PCIe 3.0 x16: 16 GB/s
Arbitration Method to grant bus access Daisy chain, polling, independent request

DMA Operation

"CPU sets up, DMA takes over, interrupts when done" - Simple way to remember DMA flow.

"Programmed I/O = CPU waits, Interrupt I/O = CPU notified, DMA = CPU freed"

Exam Preparation

Important Formulas

Number Systems
Binary to Decimal: Σbᵢ×2ⁱ
2's Complement: -X = (NOT X) + 1
Range (n-bit signed): -2ⁿ⁻¹ to 2ⁿ⁻¹-1
Performance
CPU Time = IC × CPI × Clock Time
MIPS = Clock Rate / (CPI × 10⁶)
Speedup = Old Time / New Time
Memory
AMAT = Hit Time + Miss Rate × Miss Penalty
Effective Access Time = p × T₁ + (1-p) × T₂
Pipelining
Speedup = (n × k) / (n + k - 1)
Efficiency = Speedup / k
Throughput = n / (k + n - 1) cycles

Common Question Types

1. K-Map Simplification
Given truth table or expression, simplify using K-Map. Watch for don't cares.
2. Adder/ALU Design
Design full adder using gates, or calculate carry lookahead signals.
3. Number Conversion
Convert between binary/octal/decimal/hex, perform arithmetic in 2's complement.
4. Pipeline Diagrams
Draw pipeline timing diagram, identify hazards and solutions.
5. Cache Problems
Given address sequence, calculate hits/misses for direct/set-associative cache.

Exam Tips

Time Management: Allocate time based on marks. Don't spend too long on one question.
Show Work: Even if final answer wrong, partial marks for correct method.
Diagrams: Draw clear diagrams for circuit designs, pipeline timing, cache organization.
Definitions: Know precise definitions for terms like CPI, MIPS, hazard, cache hit/miss.
Practice: Work through previous years' questions and GATE questions.
